# Security Incident Escalation Policy

## Purpose

This policy explains how suspected security incidents should be escalated.

## Security Incident Examples

Examples include:
- Phishing emails.
- Malware warnings.
- Lost or stolen laptop.
- Suspicious login alerts.
- Unauthorised access.
- Data leakage.
- MFA fatigue attacks.

## Immediate Guidance

If a user reports a suspected security issue:
1. Tell the user not to click suspicious links.
2. Tell the user not to delete evidence.
3. Ask the user to disconnect from the network if malware is suspected.
4. Ask for the time of the event and affected system.
5. Escalate to the security team.

## Sensitive Action

Security incident escalation is sensitive and may require human review.

## Ticket Handling

Create a high-priority ticket if:
- Credentials may be compromised.
- A company device is lost or stolen.
- Malware is suspected.
- Customer or company data may be exposed.
- Multiple users are affected.

## Suggested Ticket Category

Category: Security Incident  
Priority: Critical  
Urgency: Critical
