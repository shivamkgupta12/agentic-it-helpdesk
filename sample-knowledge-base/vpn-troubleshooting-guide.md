# VPN Troubleshooting Guide

## Purpose

This guide helps employees resolve common VPN connectivity problems before escalating to IT support.

## Common Symptoms

Employees may report:
- VPN disconnects frequently.
- VPN connects but internal dashboards are not accessible.
- MFA approval is not received.
- VPN client shows authentication failure.
- VPN works on one network but not another.

## Initial Checks

1. Confirm the employee has a stable internet connection.
2. Disconnect and reconnect the VPN client.
3. Restart the VPN client.
4. Confirm MFA approval was completed successfully.
5. Check whether the issue happens on another network, such as mobile hotspot.
6. Restart the laptop if the VPN client continues disconnecting.

## Internal Dashboard Access

If VPN connects but the internal dashboard is unavailable:

1. Confirm the dashboard URL is correct.
2. Clear browser cache or try a private browser window.
3. Confirm the user is connected to the corporate VPN profile.
4. Ask whether other internal systems are accessible.
5. If only one dashboard fails, route to the application support team.
6. If all internal systems fail, route to Network / VPN support.

## Escalation Criteria

Create a support ticket if:
- VPN disconnects repeatedly after restart.
- MFA succeeds but VPN still fails.
- Internal systems remain inaccessible after VPN reconnect.
- Multiple users report the same VPN problem.
- The VPN client shows certificate, device compliance, or authentication errors.

## Suggested Ticket Category

Category: Network / VPN  
Priority: Medium  
Urgency: Medium

If multiple users are affected or business-critical systems are inaccessible, priority may be High.
