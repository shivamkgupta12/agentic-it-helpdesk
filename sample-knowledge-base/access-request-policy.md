# Access Request Policy

## Purpose

This policy explains how access requests should be handled.

## Sensitive Action

Access changes are sensitive actions. The AI assistant must not grant access directly.

## Examples of Access Requests

Employees may ask for:
- Access to internal dashboards.
- Access to cloud resources.
- Access to shared drives.
- Access to admin tools.
- Access to production systems.

## Required Information

Collect:
1. Requested application or system.
2. Business justification.
3. Access level needed.
4. Duration of access.
5. Manager or system owner approval.

## Approval Requirement

Approval is required for:
- Production access.
- Admin access.
- Financial systems.
- HR systems.
- Customer data systems.
- Security tools.

## AI Assistant Rules

The AI assistant may:
- Explain the process.
- Create an access request ticket.
- Ask for required details.

The AI assistant must not:
- Grant access directly.
- Change user roles.
- Bypass approval.
- Ask users to share credentials.
